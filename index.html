What is “Legacy”?

“Legacy code is source code that relates to a no-longer supported or manufactured operating system or other computer technology” Wikipedia, Legacy Code
  • Any code as soon as it is released
  • Code you would rather rewrite than work with
  • Code maintained by someone other than the original author
“To me, legacy code is simply code without tests.” Michael C. Feathers, Effectively Working with Legacy Code

Legacy CSS - not readily understandable, doesn’t have a clear architecture, and you have to relearn it any time you want to try and change it (and there is probably a lot of it!)

Where Does It Come From?

Technologies & Methodologies Change

  • Front end development has undergone massive changes in just the last few years
  • All of this stuff has made us better, but it also contributes to legacy code

The changing landscape of front end technologies is not the lone cause of our legacy code issues

Company Culture

  • Business priorities change, deadlines get shorter, we make compromises
  • We don’t understand the impact of the technical debt we’re accruing, nor do we have a plan to pay off the debt
  • Front End Development is Easy!
  • A wide range of people with varying skill sets end up working on CSS

What’s the Big Deal?

It sucks for our customers!

  • Page performance
  • Inconsistent UI/UX

It sucks for us (developers)

  • It’s frustrating!
  • It slows you down - simple tasks take longer
  • You break stuff more often - it’s often impossible to know how your change is going to impact the rest of the system

It sucks for the business

  • Eventually this stuff affects your business strategy
  • It takes longer to get new features in front of customers
  • Good developers are unhappy, unhappy developers leave
  • It's hard to attract new developers

How do we fix our legacy problem

Invest in yourself and your team

  • Take the time to do things properly - it will pay off in huge ways
  • Take some time to reacquaint yourself with the basics
  • Coding standards and guidelines - they matter!

  • Make sure your entire team follows the same standards - spend time on training
