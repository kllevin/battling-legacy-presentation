Fighting the Good Fight

The (never ending) Legacy Battle

What is "Legacy"

  • Old code
  • Code maintained by someone other than the original author
"To me, legacy code is simply code without tests."
Michael C. Feathers, Effectively Working with Legacy Code

Where does it come from?

Methodologies change

  • Table layouts -> CSS layouts
  • OOCSS
  • preprocessors (SASS, SCSS, LESS)
  • PostCSS, CSS Modules, etc.

Changing business priorities

Company Culture

  • Front End Development is Easy!
  • A wide range of people with varying skill sets end up working on CSS

What's the big deal?

It effects customers

  • Page performance
  • Inconsistent UI/UX

It effects Developers

  • it takes longer to make simple changes
  • lack confidence in the code base
  • developer fatigue (it's frustrating!)

How do we fix our legacy problem

There is no silver bullet

We've all got different legacy codebases

It's caused by different things
